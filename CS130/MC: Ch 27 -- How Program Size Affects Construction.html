McConnell: Chapter 27 — How Program Size Affects Construction
  • As project size increases, a larger percentage of errors can usually be attributed to mistakes in requirements and design.
  • The density of error increases with increased project size.

Effect of Project Size on Productivity
  • At small sizes, the biggest influence on productivity is the skill of the individual developer.
  • As project size increases, team size and organization become greater influences on productivity.
  • Activities that grow at a more-than-linear rate as project size increases:
    • Communication
    • Planning
    • Management
    • Requirements development
    • System functional design
    • Interface design and specification
    • Architecture
    • Integration
    • Defect removal
    • System testing
    • Document production
  • When estimating project schedules, be sure to understand the additional effort needed to move from system to actual software product (all the additional polish).
  • Methodologies are important for large projects. The programmer who insists on avoiding methodologies is really only avoiding choosing one explicitly — no one can avoid using them altogether.

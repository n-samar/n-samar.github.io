McConnel: Chapter 30 — Programming Tools
  • Programming tools can increase productivity by over 50%
  • 20% of the tools tend to account for 80% of the tool usage

Source-code tools
  • Integrate Development Environments (IDEs) functionalities:
    • Compilation and error detection from within the editor
    • Integration with source-code control, build, test, and debugging tools
    • Compressed or outline views of programs (class names only or logical structures — folding)
    • Jump to definitions of classes, routines, and variables
    • Jump to all places where a class, routine, or variable is used
    • Language-specific formatting
    • Interactive help for the language being edited
    • Brace matching
    • Templates for common language constructs (e.g. for loops)
    • Smart indenting (including easily changing the indentation of a block of statements)
    • Automated code transforms or refactoring
    • Macros programmable in a familiar programming language
    • Listing of search strings so that commonly used strings don’t need to be retyped
    • Regular expressions in search-and-replace
    • Search-and-replace across a group of files
    • Editing multiple files simultaneously
    • Side-by-side diff comparisons
    • Multilevel undo
    • Merge tools
    • Interface documentation tools (ability to extract detailed programmer-interface documentation from source-code files)
    • Class hierarchy generation

Analyzing code quality
  • Syntax and semantics checkers (e.g. linters)
  • Metric reporters (how many lines of code in this routine? how many nested for loops? how many variable declarations? how many global variables? what should I refactor first?)

  • Version Control
    • Source code control
    • Dependency control
    • Project documentation versioning
  • Data Dictionaries (a database that describes all the significant data in a project)

Executable-code tools
  • Code creation
    • Compilers and linkers
    • Build tools
      • Goal: minimize the time needed to build a program using current versions of the program’s source files (e.g. make)

  • Feel free to write project specific tools and scripts
